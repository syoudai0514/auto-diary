# 音声日記PWA (Voice Diary)

iPhoneで自分の一日や出来事を口頭で話すと、AIが**自然な日本語の日記**にまとめてくれる、個人用の音声日記PWAです。作成した日記は Apple純正「ジャーナル」・Day One・クリップボード・iOS共有シートへ保存できます。

- **話すだけ** — 録音するか、キーボードの音声入力で話すだけ。
- **AIが日記化** — 事実・気持ち・解釈を分けて、読みやすい日記本文に整えます。
- **どこにでも保存** — Appleジャーナル / Day One / コピー / 共有シート。
- **PWA** — iPhoneのホーム画面に追加すればネイティブアプリのように使えます。
- **プライバシー重視** — Gemini APIはサーバー側からのみ呼び出し、APIキーはブラウザに出しません。音声・本文はサーバーに保存もログ出力もしません。
- **無料で使える** — AIには Google Gemini API の無料枠を利用しています（詳細は下記）。

「ウォーム・ペーパー」デザイン（クリーム地に墨色の文字、テラコッタのアクセント）をライト/ダーク両対応で実装しています。

## 技術構成

| 項目 | 内容 |
|---|---|
| フレームワーク | **Next.js 14**（App Router） |
| 言語 | **TypeScript**（strict） |
| スタイル | Tailwind CSS（デザイントークンをCSS変数で管理） |
| PWA | manifest + Service Worker + アイコン一式 |
| AI | Google Gemini API（無料枠あり、**サーバーサイドのみ**から呼び出し） |
| 文字起こし | `gemini-3.1-flash-lite`（音声をインラインデータとして送信、環境変数で変更可） |
| 日記生成 | `gemini-3.1-flash-lite` + Structured Output（responseSchema） |
| 認証 | パスワード + 署名付き HttpOnly Cookie |
| 下書き保存 | IndexedDB（未保存の日記のみ端末内に退避） |
| テスト | Vitest |
| デプロイ | Vercel（GitHub push で自動デプロイ） |

主なディレクトリ:

```
src/
├─ app/
│  ├─ page.tsx            … メイン画面（録音〜結果までの状態機械）
│  ├─ login/page.tsx      … ログイン
│  ├─ settings/page.tsx   … 設定（文体・保存先・外観、プロフィールへの導線）
│  ├─ profile/page.tsx    … プロフィール編集（テキスト/音声で追加、AIが整理）
│  └─ api/
│     ├─ login/route.ts          … パスワード認証・Cookie発行
│     ├─ logout/route.ts         … ログアウト
│     ├─ transcribe/route.ts     … 音声→テキスト（Gemini）
│     ├─ generate/route.ts       … テキスト→構造化日記（Gemini）
│     └─ profile/update/route.ts … プロフィール(Markdown)の統合・更新（Gemini）
├─ lib/                   … 認証・レート制限・Gemini・共有URL生成 など（純関数中心）
├─ hooks/useRecorder.ts   … マイク録音フック
├─ components/            … UIコンポーネント（インラインSVGアイコン等）
└─ middleware.ts          … 全ページ/APIの認証ガード
```

## 画面と使い方

- **ホーム**: 「録音して日記化」ボタン（マイク）、「すぐ話す（キーボード入力）」、「音声ファイルをアップロード」の3つの入り口。未保存の下書きがあれば上部にバナー、下に最近の記録。
- **録音中**: 録音時間・波形・一時停止/再開・停止・キャンセル。20分を超えると注意バナー。
- **音声ファイルから作る**: ボイスメモなどで録音済みの音声ファイルを複数選び、選んだ順に1つの日記としてまとめられます（`<input type="file" multiple>` でOSのファイル選択画面から複数選択）。ファイルは一覧表示され、個別に削除・追加ができます。
- **処理中**: 文字起こし中（複数ファイルの場合は「n/m件目」の進捗表示）→ 日記生成中。
- **結果**: タイトル/本文を編集でき、「元の文字起こし」を折りたたみ表示。チップから Appleジャーナル/Day One/各種コピー/共有/書き直す/削除。最下部の「保存する」が唯一の主要ボタン。
- **設定**: 文体（5種）・**プロフィール**（下記参照）へのリンク・標準の保存先（Appleジャーナル/Day One/クリップボード/他の日記アプリを開く/毎回選ぶ）・Day Oneのジャーナル名・外観（端末に合わせる/ライト/ダーク）・ログアウト。
- **プロフィール**: 家族構成や自分の立場などをMarkdownで書き溜める専用画面（下記参照）。

日記は次の7通りで保存できます:
1. Apple純正「ジャーナル」へ iOSショートカット経由で保存
2. Day One へ URLスキーム経由で保存
3. **他の日記アプリを開く**（コピー+Shortcuts経由で起動、下記参照）
4. タイトルのみコピー
5. 本文のみコピー
6. タイトルと本文をまとめてコピー
7. iOS共有シートから他アプリへ送信

### プロフィール（家族構成・自分について）

家族との会話など、複数人が写る/話す内容を録音すると、AIだけでは「どの発言が自分（一人称）で、周りの人物をどう呼べばいいか」を正しく判断できないことがあります。設定画面の「プロフィールを編集」から、専用の**プロフィール画面**で以下のように管理できます。

- **現在のプロフィール**: Markdown形式のテキストとして表示・直接編集できます（例: `## 家族構成` の見出しの下に `- 妻(ママ)` のように箇条書き）。
- **情報を追加**: テキスト入力、または**マイクで話して**追加できます（録音→自動で文字起こし）。入力した内容と現在のプロフィールをAIが読み取り、矛盾なく統合した新しいプロフィール全文に更新します（「AIでまとめて追加」ボタン）。

例えば

> 私は4人家族の父です。妻はママ、子どもは長男・長女と呼びます。

と最初に登録し、後日「長女が幼稚園に入りました」と音声で追加すると、AIが両方の情報を保ったまま1つのプロフィールに整理してくれます。

このプロフィールは日記生成のたびに参考情報として使われ、一人称の判定や登場人物の呼び方（「相手」ではなく「妻」「ママ」など）に反映されます。空欄のままでも動作します（その場合は「私」「相手」で判定）。

> **プライバシーについて**: プロフィールは他の設定と同様、**端末内（localStorage）にのみ保存**され、サーバーには保存されません。プロフィールの内容は「情報を追加」でAIに整理してもらうときと、日記を生成するときにだけ、そのつどGeminiへ一時的に送信されます。このアプリは**過去の日記の内容を自動的に記憶することはありません**（サーバー・端末のどちらにも日記本体を保存しないため）。プロフィールに残したい特性や気づきがあれば、都度「情報を追加」から手動で登録してください。

## ローカルで動かす

前提: **Node.js 18.18 以上**（推奨 20 / 22）。

```bash
# 1. 依存関係をインストール
npm install

# 2. 環境変数ファイルを用意
cp .env.example .env.local
#   → .env.local を開いて GEMINI_API_KEY / APP_PASSWORD / AUTH_SECRET を設定

# 3. 開発サーバー起動
npm run dev
#   → http://localhost:3000 を開く

# その他
npm run build      # 本番ビルド
npm run start      # 本番モードで起動
npm run test       # テスト実行
npm run typecheck  # 型チェック
npm run lint       # ESLint
```

> マイク録音は **https もしくは localhost** でのみ動作します（ブラウザの仕様）。iPhone実機で試すときは Vercel の https URL を使ってください。

## 環境変数

`.env.example` を参照。Vercel では「Settings → Environment Variables」に同じキーを登録します。

| キー | 必須 | 説明 | 既定値 |
|---|:---:|---|---|
| `GEMINI_API_KEY` | ✅ | Google Gemini APIのキー（無料）。**サーバー側のみ**で使用。 | — |
| `APP_PASSWORD` | ✅ | アプリを開くためのパスワード。 | — |
| `AUTH_SECRET` | ✅ | Cookie署名用のシークレット（32文字以上推奨）。 | （未設定時は `APP_PASSWORD` を使用） |
| `GEMINI_MODEL` | | 日記生成モデル。 | `gemini-3.1-flash-lite` |
| `GEMINI_TRANSCRIBE_MODEL` | | 文字起こしに使うモデル。 | `gemini-3.1-flash-lite` |
| `SESSION_DAYS` | | ログインの有効日数。 | `30` |
| `MAX_AUDIO_BYTES` | | 音声アップロード上限（バイト）。 | `15728640`（15MB） |

> `NEXT_PUBLIC_` を**絶対に付けない**でください。付けるとブラウザに露出します。`GEMINI_API_KEY` は本アプリでは常にサーバー側でのみ読み込まれます。

`AUTH_SECRET` の生成例:

```bash
openssl rand -hex 32
```

## Gemini APIキーの取得と設定（無料）

このアプリは **Google Gemini API** を使っています。個人利用の範囲であれば**無料枠**だけで動かせます（クレジットカード登録不要）。

1. <https://aistudio.google.com/apikey> にアクセスし、Googleアカウントでログイン。
2. 「**Create API key**」（APIキーを作成）をクリック。
3. 新しいプロジェクトを作るか、既存のプロジェクトを選ぶ（個人利用ならどちらでもOK。迷ったら「Create API key in new project」）。
4. 発行された `AIza...` から始まるキーをコピー。
5. ローカルでは `.env.local` の `GEMINI_API_KEY=` に貼り付け。Vercelでは環境変数に登録（後述）。

> ⚠️ **無料枠の注意点**: Google AI Studio経由の無料利用では、送信した内容（音声・テキスト）が Google のサービス改善に利用される場合があります（有料版・Vertex AI経由では原則利用されません）。日記は個人的な内容を含むため、この点を理解した上でご利用ください。気になる場合は有料枠（Billingを有効化）に切り替えることもできます。
>
> 無料枠にはリクエスト数の**レート制限**があります（1分あたり・1日あたりの上限）。個人が1日に数回日記をつける用途では十分ですが、短時間に何度も試すと制限にかかることがあります（その場合はアプリ側にも簡易レート制限があるため、少し待ってから再試行してください）。
>
> **モデル名について**: Googleは無料枠のモデル名・レート制限を随時変更します（例: `gemini-2.0-flash`系は2026-06-01付けで廃止）。429エラーが頻発する場合は、[Google AI Studioの使用状況ページ](https://ai.dev/rate-limit)で現在のモデルごとの制限（RPM/RPD）を確認し、より枠の広いモデル名を `GEMINI_MODEL` / `GEMINI_TRANSCRIBE_MODEL` に設定してください。

利用モデルは環境変数で差し替えられます。既定の `gemini-3.1-flash-lite` は無料枠が広く、日本語の文字起こし・日記生成の両方に十分な性能があります。

## GitHub連携手順

**リポジトリは Private を前提**にしています（APIキーは環境変数で扱うためコードには入りませんが、個人アプリとしてPrivate推奨）。

1. GitHubで新しい **Private** リポジトリを作成（例: `auto-diary`）。
2. ローカルで初回コミット & プッシュ:

   ```bash
   git init                # まだの場合
   git add .
   git commit -m "feat: 音声日記PWAの初期実装"
   git branch -M main
   git remote add origin git@github.com:<あなた>/auto-diary.git
   git push -u origin main
   ```

3. `.gitignore` により `.env` / `.env.local` はコミットされません（**キーが漏れない**ようになっています）。
4. 以降、GitHubへ push すると Vercel が自動でデプロイします（次項で連携）。

## Vercelデプロイ手順

1. <https://vercel.com/> に GitHub アカウントでログイン。
2. 「**Add New… → Project**」で先ほどの GitHub リポジトリを **Import**。
3. Framework Preset は自動で **Next.js** が選ばれます。そのまま **Deploy** せず、先に環境変数を設定します。
4. 「**Environment Variables**」で以下を追加（[環境変数](#環境変数)参照）:
   - `GEMINI_API_KEY`
   - `APP_PASSWORD`
   - `AUTH_SECRET`
   - （任意）`GEMINI_MODEL` / `GEMINI_TRANSCRIBE_MODEL` / `SESSION_DAYS` / `MAX_AUDIO_BYTES`
5. 「**Deploy**」を実行。数十秒で `https://<プロジェクト名>.vercel.app` が発行されます。
6. 以後、`main` ブランチへ push するたびに**自動で再デプロイ**されます。

> 文字起こし/生成APIは処理に時間がかかるため `maxDuration = 60`（秒）を指定しています。無料プランでも動作しますが、長い音声はタイムアウトすることがあります。その場合は録音を短く分けてください。

## iPhoneのホーム画面に追加する手順

1. iPhoneの **Safari** で、Vercelで発行された `https://…vercel.app` を開く。
2. パスワード（`APP_PASSWORD`）でログイン。
3. 画面下の **共有ボタン**（□に↑）をタップ。
4. 「**ホーム画面に追加**」をタップ。
5. 名前（例: 音声日記）を確認して「**追加**」。
6. ホーム画面のアイコンから起動すると、アドレスバーのない**フルスクリーン（standalone）**で使えます。

> 初回のマイク使用時に許可を求められます。「許可」を選んでください。セーフエリア（ノッチ/ホームインジケータ）にもレイアウトが対応しています。

## Appleジャーナル保存用ショートカットの作り方

アプリの「Appleジャーナルに保存」ボタンは、iOSの**ショートカット**「**音声日記を保存**」を呼び出します。このショートカットを一度だけ作っておきましょう。**順番どおりタップするだけ**でできます。

> 用意するもの: iPhone（iOS 17.2以降・「ジャーナル」アプリが使えるもの）と「ショートカット」アプリ。

### ステップ（1アクションずつ）

1. **「ショートカット」アプリを開く。**（青いアイコン）
2. 右上の **＋（プラス）** をタップして、新しいショートカットを作る。
3. 上の名前の部分をタップして、名前を **`音声日記を保存`** にする。（※アプリと同じ名前にするのが超重要）
4. 「**アクションを追加**」をタップ。
5. 検索欄に「**ショートカットの入力**」と入れて、**「ショートカットの入力を受け取る」**（Receive Input）を選ぶ。
   - 「受け取る内容」に **テキスト** が含まれるようにする。
6. もう一度「**アクションを追加**」をタップ。検索欄に「**辞書**」と入れて、**「入力から辞書を取得」**（Get Dictionary from Input）を選ぶ。
   - このアクションの入力が、5でうけとった「**ショートカットの入力**」になるようにする。
7. また「**アクションを追加**」。「**辞書の値**」と検索して、**「辞書の値を取得」**（Get Dictionary Value）を選ぶ。
   - 「キー」に **`title`** と入力する。（これでタイトルが取り出せる）
   - できた結果に、右にある `>` などから名前をつけられるなら **「タイトル」** と名づけておくと分かりやすい。
8. もう1つ「**辞書の値を取得**」を追加。今度は「キー」に **`body`** と入力する。（これで本文が取り出せる）
   - 名前を **「本文」** にしておく。
9. 「**アクションを追加**」→「**ジャーナル**」と検索して、**「ジャーナル項目を作成」**（Create Journal Entry）を選ぶ。
   - 本文の欄に、8でつくった **「本文」** を入れる。
   - タイトルの欄があれば、7でつくった **「タイトル」** を入れる。
10. さいごに「**アクションを追加**」→「**通知**」と検索して、**「通知を表示」** を選び、文章に **`日記を保存しました`** と書く。（保存できたか分かるように）
11. 右上の **完了** をタップして保存。

これで完成です。アプリの結果画面で「Appleジャーナルに保存」を押すと、このショートカットが起動して、ジャーナルに新しい記録が追加されます。

### 渡されるデータについて（作る人向けメモ）

アプリは次のURLでショートカットを起動します:

```
shortcuts://run-shortcut?name=音声日記を保存&input=text&text={URLエンコード済みJSON}
```

`text` の中身（JSON）は次の形です:

```json
{
  "title": "タイトル",
  "body": "本文",
  "tags": ["タグ"],
  "createdAt": "2026-07-11T09:30:00.000Z"
}
```

ショートカットの手順は、この `text`（＝ショートカットの入力）を辞書に変換し、`title` と `body` を取り出してジャーナルに保存する、というものです。

### うまくいかないとき（長すぎる場合の代替）

日記がとても長いと、URLが長すぎてショートカットが起動できないことがあります。その場合アプリは自動で**内容をクリップボードにコピー**し、「貼り付けてください」と案内します。ショートカット側で「クリップボードを取得」してから辞書に変換するようにしておくと、この方法でも保存できます。

## Day One連携

「Day Oneに保存」ボタンは Day One の URLスキームを使います:

```
dayone://post?entry={本文}&journal={ジャーナル名}&tags={タグ}
```

- 日本語・改行・記号は正しく **URLエンコード**して渡します。
- タイトルは本文の先頭に見出しとして付けて渡します。
- 保存先ジャーナル名は**設定画面**で指定できます（空欄なら既定ジャーナル）。
- **Day Oneが入っていない場合**でも、事前に本文を**クリップボードへコピー**しているので、手動で貼り付けられます。

> App Storeから「Day One」をインストールし、一度アプリを開いておくとURLスキームが有効になります。

## 他の日記アプリとの連携（URLスキームが無いアプリ向け）

Apple純正ジャーナルやDay One以外にも、iPhoneには数多くの日記アプリがありますが、その多くは**外部アプリからの自動投稿に対応していません**（独自のURLスキーム・共有シート対応・Shortcutsアクションのいずれも持たない）。

そのようなアプリでも、結果画面の「**他のアプリを開く**」ボタン（または設定で保存先を「他の日記アプリを開く」に設定）を使うと、次の流れで手間を最小限にできます。

1. アプリが自動でタイトルと本文を**クリップボードにコピー**
2. あらかじめ作っておいたショートカット「**日記アプリを開く**」を自動起動
3. ショートカットが対象の日記アプリを開く
4. あなたが日記アプリ側で新規作成し、**貼り付け**て保存

### 「日記アプリを開く」ショートカットの作り方

1. iPhoneの「**ショートカット**」アプリを開く
2. 右上の **＋** をタップして新しいショートカットを作る
3. 名前を **`日記アプリを開く`** にする（アプリ側の設定と同じ名前にすること）
4. 「**アクションを追加**」→「**アプリを開く**」を検索して選ぶ
5. 開くアプリとして、普段使っている日記アプリ（例: シンプル日記など）を選ぶ
6. 右上の **完了** で保存

これだけです。以後、結果画面の「他のアプリを開く」を押すと、本文がコピーされた状態でそのアプリが開くので、新規作成して貼り付けるだけで保存できます。

> 対象アプリが独自のURLスキームや共有シートに対応しているか不明な場合は、まず「共有」ボタンから共有シートを開き、アプリの候補に出てくるか確認してみてください。出てくれば、そちらの方がさらに手軽です（コピー操作が不要になります）。iOSの「ショートカット」アプリの「アプリ」タブでアプリ名を検索し、そのアプリ専用のアクション（「エントリーを作成」等の具体的な名前のもの）が見つかれば、より直接的な連携ができる場合もあります。

## セキュリティ上の注意

このアプリは個人利用向けですが、公開URLを知られても第三者に使われないよう配慮しています。

- **認証**: `APP_PASSWORD` によるログイン。成功時に**署名付き HttpOnly Cookie**（HMAC-SHA256）を発行。`middleware.ts` で全ページ/APIを保護し、各APIでも多層的に検証します。
- **APIキー秘匿**: `GEMINI_API_KEY` は**サーバーサイドのみ**で読み込み、クライアントへは一切送りません。
- **保存しない**: 音声はサーバー/Vercelに**永続保存しません**（文字起こし後に破棄）。音声・日記本文を**サーバーログに出力しません**（エラーログにも本文を含めません）。
- **端末内のみ**: 未保存の下書きだけを **IndexedDB** に保存します。パスワードやAPIキーは localStorage 等に保存しません。
- **レート制限**: ログイン・文字起こし・生成に簡易レート制限（トークンバケット）を実装し、連続呼び出しを防ぎます。
- **入力サイズ制限**: 音声サイズ（既定15MB）・文字起こし長（2万文字）・パスワード長などに上限を設定。
- **CSRF/XSS**: Cookieは `SameSite=Lax` / `HttpOnly` / 本番は `Secure`。表示はReactの自動エスケープに委ね、`dangerouslySetInnerHTML` は不使用。
- **注意点**: レート制限はインメモリのため、サーバーレスの複数インスタンス環境では厳密ではありません。より強固にするには Upstash Redis 等の外部ストアを推奨します（[今後の拡張案](#今後の拡張案)参照）。
- **Gemini無料枠のデータ利用について**: Google AI Studio経由の無料利用は、送信内容がサービス改善に使われる場合があります。日記という個人的な内容を扱うため、この前提を理解した上でご利用ください。気になる場合はGeminiの有料枠（Billing有効化）への切り替えをご検討ください。

> ⚠️ `APP_PASSWORD` と `AUTH_SECRET` は十分に長いランダム文字列にし、`.env*` を**絶対にコミットしない**でください（`.gitignore`済み）。

## 今後の拡張案

- **日記の本保存**: 現状は外部アプリ保存が中心。将来は暗号化した端末内DBやクラウド（Supabase等）へ日記本体を保存し、一覧・検索・カレンダー表示。
- **分割文字起こし**: 15MB超の長時間録音を自動分割してつなぐ処理。
- **有料枠への切り替え**: プライバシーを重視する場合、Geminiの有料枠（Billing有効化）や、学習データに使われない他のAPIへの切り替え対応。
- **日記からのプロフィール更新提案**: 生成された日記から「プロフィールに追加しますか？」と提案する半自動フロー（現状は手動で「情報を追加」する運用。日記本体を保存しない設計を保ったまま、ユーザーの明示的な承認を経て追加する形を想定）。
- **話者の口ぐせ学習**: 過去の日記から文体を学習し、よりその人らしい文章に。
- **タグの自動整理 / 感情の可視化**: `feelings`・`tags` を使った振り返りダッシュボード。
- **Web Push リマインダー**: 「今日の日記を書きましょう」通知。
- **多要素/パスキー認証**: WebAuthn 対応。
- **分散レート制限**: Upstash Redis などで正確なレート制限・不正アクセス検知。
- **オフライン録音キュー**: 電波が無い場所で録音 → 復帰時に自動で文字起こし。
- **多言語対応**: 文字起こし言語の自動判定と英語日記など。
