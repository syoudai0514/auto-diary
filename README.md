# 音声日記PWA (Voice Diary)

iPhoneで自分の一日や出来事を口頭で話すと、AIが**自然な日本語の日記**にまとめてくれる、個人用の音声日記PWAです。作成した日記は Apple純正「ジャーナル」・Day One・クリップボード・iOS共有シートへ保存できます。

- **話すだけ** — 録音するか、キーボードの音声入力で話すだけ。
- **AIが日記化** — 事実・気持ち・解釈を分けて、読みやすい日記本文に整えます。
- **どこにでも保存** — Appleジャーナル / Day One / コピー / 共有シート。
- **PWA** — iPhoneのホーム画面に追加すればネイティブアプリのように使えます。
- **プライバシー重視** — OpenAI APIはサーバー側からのみ呼び出し、APIキーはブラウザに出しません。音声・本文はサーバーに保存もログ出力もしません。

「ウォーム・ペーパー」デザイン（クリーム地に墨色の文字、テラコッタのアクセント）をライト/ダーク両対応で実装しています。

## 技術構成

| 項目 | 内容 |
|---|---|
| フレームワーク | **Next.js 14**（App Router） |
| 言語 | **TypeScript**（strict） |
| スタイル | Tailwind CSS（デザイントークンをCSS変数で管理） |
| PWA | manifest + Service Worker + アイコン一式 |
| AI | OpenAI API（**サーバーサイドのみ**から呼び出し） |
| 文字起こし | `gpt-4o-mini-transcribe`（環境変数で変更可） |
| 日記生成 | `gpt-4o-mini` + Structured Output（JSON Schema） |
| 認証 | パスワード + 署名付き HttpOnly Cookie |
| 下書き保存 | IndexedDB（未保存の日記のみ端末内に退避） |
| テスト | Vitest |
| デプロイ | Vercel（GitHub push で自動デプロイ） |

主なディレクトリ:

```
src/
├─ app/
│  ├─ page.tsx            … メイン画面（録音〜結果までの状態機械）
│  ├─ login/page.tsx      … ログイン
│  ├─ settings/page.tsx   … 設定（文体・保存先・外観）
│  └─ api/
│     ├─ login/route.ts       … パスワード認証・Cookie発行
│     ├─ logout/route.ts      … ログアウト
│     ├─ transcribe/route.ts  … 音声→テキスト（OpenAI）
│     └─ generate/route.ts    … テキスト→構造化日記（OpenAI）
├─ lib/                   … 認証・レート制限・OpenAI・共有URL生成 など（純関数中心）
├─ hooks/useRecorder.ts   … マイク録音フック
├─ components/            … UIコンポーネント（インラインSVGアイコン等）
└─ middleware.ts          … 全ページ/APIの認証ガード
```

## 画面と使い方

- **ホーム**: 「録音して日記化」ボタン（マイク）と「すぐ話す（キーボード入力）」ボタン。未保存の下書きがあれば上部にバナー、下に最近の記録。
- **録音中**: 録音時間・波形・一時停止/再開・停止・キャンセル。20分を超えると注意バナー。
- **処理中**: 文字起こし中 → 日記生成中。
- **結果**: タイトル/本文を編集でき、「元の文字起こし」を折りたたみ表示。チップから Appleジャーナル/Day One/各種コピー/共有/書き直す/削除。最下部の「保存する」が唯一の主要ボタン。
- **設定**: 文体（5種）・標準の保存先・Day Oneのジャーナル名・外観（端末に合わせる/ライト/ダーク）・ログアウト。

日記は次の6通りで保存できます:
1. Apple純正「ジャーナル」へ iOSショートカット経由で保存
2. Day One へ URLスキーム経由で保存
3. タイトルのみコピー
4. 本文のみコピー
5. タイトルと本文をまとめてコピー
6. iOS共有シートから他アプリへ送信

## ローカルで動かす

前提: **Node.js 18.18 以上**（推奨 20 / 22）。

```bash
# 1. 依存関係をインストール
npm install

# 2. 環境変数ファイルを用意
cp .env.example .env.local
#   → .env.local を開いて OPENAI_API_KEY / APP_PASSWORD / AUTH_SECRET を設定

# 3. 開発サーバー起動
npm run dev
#   → http://localhost:3000 を開く

# その他
npm run build      # 本番ビルド
npm run start      # 本番モードで起動
npm run test       # テスト実行
npm run typecheck  # 型チェック
npm run lint       # ESLint
```

> マイク録音は **https もしくは localhost** でのみ動作します（ブラウザの仕様）。iPhone実機で試すときは Vercel の https URL を使ってください。

## 環境変数

`.env.example` を参照。Vercel では「Settings → Environment Variables」に同じキーを登録します。

| キー | 必須 | 説明 | 既定値 |
|---|:---:|---|---|
| `OPENAI_API_KEY` | ✅ | OpenAIのAPIキー。**サーバー側のみ**で使用。 | — |
| `APP_PASSWORD` | ✅ | アプリを開くためのパスワード。 | — |
| `AUTH_SECRET` | ✅ | Cookie署名用のシークレット（32文字以上推奨）。 | （未設定時は `APP_PASSWORD` を使用） |
| `OPENAI_MODEL` | | 日記生成モデル。 | `gpt-4o-mini` |
| `TRANSCRIBE_MODEL` | | 文字起こしモデル。 | `gpt-4o-mini-transcribe` |
| `SESSION_DAYS` | | ログインの有効日数。 | `30` |
| `MAX_AUDIO_BYTES` | | 音声アップロード上限（バイト）。 | `26214400`（25MB） |

> `NEXT_PUBLIC_` を**絶対に付けない**でください。付けるとブラウザに露出します。`OPENAI_API_KEY` は本アプリでは常にサーバー側でのみ読み込まれます。

`AUTH_SECRET` の生成例:

```bash
openssl rand -hex 32
```

## OpenAI APIキーの取得と設定

1. <https://platform.openai.com/> にログイン。
2. 右上のメニュー →「**API keys**」→「**Create new secret key**」。
3. 表示された `sk-...` をコピー（**一度しか表示されません**）。
4. ローカルでは `.env.local` の `OPENAI_API_KEY=` に貼り付け。Vercelでは環境変数に登録（後述）。
5. 支払い方法の登録が必要な場合があります（Billing）。使用量に応じた**上限（Usage limits）**を設定しておくと安心です。

利用モデルは環境変数で差し替えられます。コストを抑えたい場合は既定の `gpt-4o-mini` / `gpt-4o-mini-transcribe` のままがおすすめです。

## GitHub連携手順

**リポジトリは Private を前提**にしています（APIキーは環境変数で扱うためコードには入りませんが、個人アプリとしてPrivate推奨）。

1. GitHubで新しい **Private** リポジトリを作成（例: `auto-diary`）。
2. ローカルで初回コミット & プッシュ:

   ```bash
   git init                # まだの場合
   git add .
   git commit -m "feat: 音声日記PWAの初期実装"
   git branch -M main
   git remote add origin git@github.com:<あなた>/auto-diary.git
   git push -u origin main
   ```

3. `.gitignore` により `.env` / `.env.local` はコミットされません（**キーが漏れない**ようになっています）。
4. 以降、GitHubへ push すると Vercel が自動でデプロイします（次項で連携）。

## Vercelデプロイ手順

1. <https://vercel.com/> に GitHub アカウントでログイン。
2. 「**Add New… → Project**」で先ほどの GitHub リポジトリを **Import**。
3. Framework Preset は自動で **Next.js** が選ばれます。そのまま **Deploy** せず、先に環境変数を設定します。
4. 「**Environment Variables**」で以下を追加（[環境変数](#環境変数)参照）:
   - `OPENAI_API_KEY`
   - `APP_PASSWORD`
   - `AUTH_SECRET`
   - （任意）`OPENAI_MODEL` / `TRANSCRIBE_MODEL` / `SESSION_DAYS` / `MAX_AUDIO_BYTES`
5. 「**Deploy**」を実行。数十秒で `https://<プロジェクト名>.vercel.app` が発行されます。
6. 以後、`main` ブランチへ push するたびに**自動で再デプロイ**されます。

> 文字起こし/生成APIは処理に時間がかかるため `maxDuration = 60`（秒）を指定しています。無料プランでも動作しますが、長い音声はタイムアウトすることがあります。その場合は録音を短く分けてください。

## iPhoneのホーム画面に追加する手順

1. iPhoneの **Safari** で、Vercelで発行された `https://…vercel.app` を開く。
2. パスワード（`APP_PASSWORD`）でログイン。
3. 画面下の **共有ボタン**（□に↑）をタップ。
4. 「**ホーム画面に追加**」をタップ。
5. 名前（例: 音声日記）を確認して「**追加**」。
6. ホーム画面のアイコンから起動すると、アドレスバーのない**フルスクリーン（standalone）**で使えます。

> 初回のマイク使用時に許可を求められます。「許可」を選んでください。セーフエリア（ノッチ/ホームインジケータ）にもレイアウトが対応しています。

## Appleジャーナル保存用ショートカットの作り方

アプリの「Appleジャーナルに保存」ボタンは、iOSの**ショートカット**「**音声日記を保存**」を呼び出します。このショートカットを一度だけ作っておきましょう。**順番どおりタップするだけ**でできます。

> 用意するもの: iPhone（iOS 17.2以降・「ジャーナル」アプリが使えるもの）と「ショートカット」アプリ。

### ステップ（1アクションずつ）

1. **「ショートカット」アプリを開く。**（青いアイコン）
2. 右上の **＋（プラス）** をタップして、新しいショートカットを作る。
3. 上の名前の部分をタップして、名前を **`音声日記を保存`** にする。（※アプリと同じ名前にするのが超重要）
4. 「**アクションを追加**」をタップ。
5. 検索欄に「**ショートカットの入力**」と入れて、**「ショートカットの入力を受け取る」**（Receive Input）を選ぶ。
   - 「受け取る内容」に **テキスト** が含まれるようにする。
6. もう一度「**アクションを追加**」をタップ。検索欄に「**辞書**」と入れて、**「入力から辞書を取得」**（Get Dictionary from Input）を選ぶ。
   - このアクションの入力が、5でうけとった「**ショートカットの入力**」になるようにする。
7. また「**アクションを追加**」。「**辞書の値**」と検索して、**「辞書の値を取得」**（Get Dictionary Value）を選ぶ。
   - 「キー」に **`title`** と入力する。（これでタイトルが取り出せる）
   - できた結果に、右にある `>` などから名前をつけられるなら **「タイトル」** と名づけておくと分かりやすい。
8. もう1つ「**辞書の値を取得**」を追加。今度は「キー」に **`body`** と入力する。（これで本文が取り出せる）
   - 名前を **「本文」** にしておく。
9. 「**アクションを追加**」→「**ジャーナル**」と検索して、**「ジャーナル項目を作成」**（Create Journal Entry）を選ぶ。
   - 本文の欄に、8でつくった **「本文」** を入れる。
   - タイトルの欄があれば、7でつくった **「タイトル」** を入れる。
10. さいごに「**アクションを追加**」→「**通知**」と検索して、**「通知を表示」** を選び、文章に **`日記を保存しました`** と書く。（保存できたか分かるように）
11. 右上の **完了** をタップして保存。

これで完成です。アプリの結果画面で「Appleジャーナルに保存」を押すと、このショートカットが起動して、ジャーナルに新しい記録が追加されます。

### 渡されるデータについて（作る人向けメモ）

アプリは次のURLでショートカットを起動します:

```
shortcuts://run-shortcut?name=音声日記を保存&input=text&text={URLエンコード済みJSON}
```

`text` の中身（JSON）は次の形です:

```json
{
  "title": "タイトル",
  "body": "本文",
  "tags": ["タグ"],
  "createdAt": "2026-07-11T09:30:00.000Z"
}
```

ショートカットの手順は、この `text`（＝ショートカットの入力）を辞書に変換し、`title` と `body` を取り出してジャーナルに保存する、というものです。

### うまくいかないとき（長すぎる場合の代替）

日記がとても長いと、URLが長すぎてショートカットが起動できないことがあります。その場合アプリは自動で**内容をクリップボードにコピー**し、「貼り付けてください」と案内します。ショートカット側で「クリップボードを取得」してから辞書に変換するようにしておくと、この方法でも保存できます。

## Day One連携

「Day Oneに保存」ボタンは Day One の URLスキームを使います:

```
dayone://post?entry={本文}&journal={ジャーナル名}&tags={タグ}
```

- 日本語・改行・記号は正しく **URLエンコード**して渡します。
- タイトルは本文の先頭に見出しとして付けて渡します。
- 保存先ジャーナル名は**設定画面**で指定できます（空欄なら既定ジャーナル）。
- **Day Oneが入っていない場合**でも、事前に本文を**クリップボードへコピー**しているので、手動で貼り付けられます。

> App Storeから「Day One」をインストールし、一度アプリを開いておくとURLスキームが有効になります。

## セキュリティ上の注意

このアプリは個人利用向けですが、公開URLを知られても第三者に使われないよう配慮しています。

- **認証**: `APP_PASSWORD` によるログイン。成功時に**署名付き HttpOnly Cookie**（HMAC-SHA256）を発行。`middleware.ts` で全ページ/APIを保護し、各APIでも多層的に検証します。
- **APIキー秘匿**: `OPENAI_API_KEY` は**サーバーサイドのみ**で読み込み、クライアントへは一切送りません。
- **保存しない**: 音声はサーバー/Vercelに**永続保存しません**（文字起こし後に破棄）。音声・日記本文を**サーバーログに出力しません**（エラーログにも本文を含めません）。
- **端末内のみ**: 未保存の下書きだけを **IndexedDB** に保存します。パスワードやAPIキーは localStorage 等に保存しません。
- **レート制限**: ログイン・文字起こし・生成に簡易レート制限（トークンバケット）を実装し、連続呼び出しを防ぎます。
- **入力サイズ制限**: 音声サイズ（既定25MB）・文字起こし長（2万文字）・パスワード長などに上限を設定。
- **CSRF/XSS**: Cookieは `SameSite=Lax` / `HttpOnly` / 本番は `Secure`。表示はReactの自動エスケープに委ね、`dangerouslySetInnerHTML` は不使用。
- **注意点**: レート制限はインメモリのため、サーバーレスの複数インスタンス環境では厳密ではありません。より強固にするには Upstash Redis 等の外部ストアを推奨します（[今後の拡張案](#今後の拡張案)参照）。

> ⚠️ `APP_PASSWORD` と `AUTH_SECRET` は十分に長いランダム文字列にし、`.env*` を**絶対にコミットしない**でください（`.gitignore`済み）。

## 今後の拡張案

- **日記の本保存**: 現状は外部アプリ保存が中心。将来は暗号化した端末内DBやクラウド（Supabase等）へ日記本体を保存し、一覧・検索・カレンダー表示。
- **分割文字起こし**: 25MB超の長時間録音を自動分割してつなぐ処理。
- **話者の口ぐせ学習**: 過去の日記から文体を学習し、よりその人らしい文章に。
- **タグの自動整理 / 感情の可視化**: `feelings`・`tags` を使った振り返りダッシュボード。
- **Web Push リマインダー**: 「今日の日記を書きましょう」通知。
- **多要素/パスキー認証**: WebAuthn 対応。
- **分散レート制限**: Upstash Redis などで正確なレート制限・不正アクセス検知。
- **オフライン録音キュー**: 電波が無い場所で録音 → 復帰時に自動で文字起こし。
- **多言語対応**: 文字起こし言語の自動判定と英語日記など。
