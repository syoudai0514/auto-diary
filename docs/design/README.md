# Handoff: 音声日記PWA (Voice Diary PWA) — iPhone UI/UX

## Overview
個人用の音声日記PWA。ユーザーは録音ボタンをタップして日々の出来事を話し、アプリが自動で文字起こし・日記本文を生成する。生成された日記は確認・編集・外部アプリ(Appleジャーナル/Day One)への保存・コピーができる。iPhone縦画面・片手操作・ホーム画面追加後にネイティブアプリのように見えることを最優先に設計。

## About the Design Files
このバンドル内のHTMLファイルは **HTMLで作成したデザインリファレンス（プロトタイプ）** であり、見た目と挙動の意図を示すものであって、本番コードとしてそのまま使うものではありません。実装タスクは、対象コードベースの既存環境（React Native、Next.js PWA、SwiftUI等、プロジェクトが採用しているスタック）の作法に沿って、このHTMLデザインを**再構築**することです。まだ環境が決まっていない場合は、本READMEの推奨（Next.js + TypeScript + Tailwind CSS、PWA構成）を採用してください。

## Fidelity
**High-fidelity（ハイファイ）**。色・タイポグラフィ・余白・角丸・アニメーションのタイミングは最終値として確定しています。既存のライブラリ/パターンを使いつつ、ピクセルレベルで忠実に再現してください。

## 採用ビジュアル方向性
3案（ウォーム・ペーパー／ソフト・スタジオ／クワイエット・スレート）を比較検討の上、**「ウォーム・ペーパー」** を採用。クリーム地に墨色の文字、テラコッタのアクセントで、内省的・温かみのある日記体験を優先。紫を基調とした典型的なAI UIを避け、過剰なグラデーション・絵文字・多用されたカードを排除している。

## Screens / Views

### 1. オンボーディング (Onboarding)
- **目的**: アプリの価値提案を1画面で伝え、次のステップ（マイク権限）へ誘導する。
- **レイアウト**: 縦積み中央揃え。上部パディング130px、左右28px。アイコン(88×88, 角丸26px, 背景アクセント色)→見出し→本文→CTAボタン→補足テキストの順。
- **コンポーネント**:
  - 見出し: 23px / 700 / line-height 1.4
  - 本文: 15px / 400 / line-height 1.85、最大幅270px
  - CTA「はじめる」: 高さ56px、幅100%、角丸28px(pill)、背景アクセント色、文字17px/600/白、box-shadow あり
  - 補足: 12px、textTertiary色

### 2. マイク権限拒否 (Permission Denied)
- 警告アイコン(80×80、角丸24px、背景warningSoft)+見出し20px/700+本文14px+CTA「設定を開く」(52px高、accent背景)+テキストリンク「後で」(44pt以上のタップ領域を確保)。

### 3. ホーム (Home)
- **レイアウト**: 上部ヘッダー(タイトル「日記」24px/700 + 設定アイコン36×36円形)。未保存下書きがある場合、ヘッダー直下にバナー(角丸16px、背景warningSoft、アイコン+テキスト+「再開」「破棄」ボタン)。中央はスクロール可能なエントリー一覧(日付・秒数12px + タイトル16px/600、行間は1px境界線区切り、行高44pt以上)。空状態は中央にプレースホルダーテキストのみ。
- **フッター（親指到達域）**: 画面下部固定、上方向にbg色でフェードするグラデーション、中央に録音FAB(76×76、角丸38px、accent背景、box-shadow)。タップでscale(1.08)。

### 4. 録音中 / 一時停止 / 長すぎる録音 (Recording / Paused / Too Long)
- **レイアウト**: 上部に警告バナー(20分目安超過時のみ)。中央に録音インジケータ(赤ドット10×10 + テキスト、録音中はパルスアニメーション、一時停止中は灰色ドット静止)。大きなタイマー表示(52px/700/tabular-nums)。中央下に波形(24本のバー、幅4px、高さ12-38pxをランダム、色accent、各バーが位相をずらしてscaleYアニメーション)。
- **コントロール**: 一時停止/再生(56×56、card背景)・停止(76×76、recording色背景、最重要操作として最大)・キャンセル(56×56、card背景)の3ボタンを28pxギャップで横並び。

### 5. 文字起こし中 / 日記生成中 (Processing)
- 同一レイアウトを2状態で共用。中央に円弧スピナー(64×64、3px border、accent色でtop-color、1s linearで360°回転)+タイトル18px/700+本文13.5px+「キャンセル」テキストボタン。派手な演出やパーティクルは使用しない。

### 6. 通信エラー (Error)
- エラーアイコン(80×80、errorSoft背景)+見出し20px/700「生成に失敗しました」+本文「文字起こしは保存されています」+CTA「再試行」(accent背景)+「戻る」リンク。

### 7. 空の録音 (Empty Recording)
- 2秒未満、または無音判定時。アイコン+見出し「音声が検出されませんでした」+CTA「もう一度録音する」+「ホームへ」リンク。

### 8. 日記生成完了 (Result / Diary Complete)
- **レイアウト**: 上部に戻るボタン+録音時間ラベル。スクロール本文エリアにタイトル(21px/700)+本文(15.5px/1.95)。その下に「元の文字起こし」の折りたたみ（デフォルト閉、タップでchevron回転200ms、展開時はcard背景の枠内に表示）。
- **二次操作**: 横スクロールのチップ列（高さ36px、card背景、border 1px）に以下を配置、**主要保存ボタンより視覚的に弱く**:
  1. Appleジャーナルに保存
  2. Day Oneに保存
  3. タイトルをコピー
  4. 本文をコピー
  5. 全文をコピー
  6. 書き直す
  7. 削除（error色でテキスト表示、破壊的操作として区別）
- **主要CTA**: 画面最下部に「保存する」ボタン1つのみ(56px高、accent背景、17px/700、box-shadow)。これが**唯一の強調ボタン**。

### 9. 保存完了トースト (Saved Toast)
- 画面下部中央にオーバーレイ(bottom:110px)、チェックアイコン(20×20、success背景、pop-inアニメーション320ms)+「保存しました」13.5px/600。1.5秒後に自動フェードアウトしホームへ遷移。

### 10. 未保存下書きあり (Draft Pending)
- ホーム画面のバナーとして表現（上記参照）。生成完了後、保存せず離脱した場合に発生。

## Interactions & Behavior
- **録音フロー**: ホーム → 録音FABタップ → 録音中(タイマー開始) → 一時停止⇄再開 → 停止 → (2秒未満なら「空の録音」へ) → 文字起こし中 → 生成中 → 結果画面（失敗時はエラー画面、再試行で生成中に戻る）。
- **保存**: 結果画面の「保存する」→ トースト表示 → ホームへ遷移しエントリー一覧に反映。
- **画面遷移アニメーション**: フェード+8px上方向スライド、200ms ease-out。
- **Reduce Motion対応**: `prefers-reduced-motion: reduce` 検知時、上記すべてのループ・トランジション（波形、パルス、スピナー、画面遷移）を無効化し瞬時切り替えにする。
- **キーボード表示時**: タイトル/本文の編集中は主要保存ボタンをキーボード直上に追従させ、隠れないようにする(sticky footer or visualViewport API)。

## State Management
状態機械として実装:
```
idle (home) → recording ⇄ paused → transcribing → generating → result
                    ↓ (stop, <2s)                        ↓ (fail)
                  empty                                error → (retry) → generating
```
主な状態変数: `screen`, `isPaused`, `elapsedSeconds`, `draftPending`, `transcriptOpen`, `savedToastVisible`, `homeEntries[]`, `theme(light/dark)`, `reducedMotion`。

ダークモードは単純反転ではなく同一色相を保持したまま明度を反転。アクセント/録音色はダーク側で明度+8〜10%。

### タイポグラフィ
- フォント: `'Noto Sans JP', -apple-system, system-ui, sans-serif`
- 見出し: 20–24px / 700 / line-height 1.4
- 小見出し: 16–17px / 600 / line-height 1.5
- 本文: 15–16px / 400 / line-height 1.85–1.95
- 補足: 12–13px / 400 / line-height 1.6
- タイマー等の数値: `font-variant-numeric: tabular-nums`

### 余白 (4pt base)
4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48px。画面外周20–24px、セクション間32–40px、リスト行16px。

### 角丸
チップ12–16px／カード・バナー14–16px／ボタンはpill(height/2)／デバイスシート24–26px。

### 影
通常カードは影なし〜1px境界線のみ。主要CTAのみ `0 10px 22px rgba(accent, 0.3)`。トースト `0 12px 28px rgba(0,0,0,0.22)`。多層シャドウの乱用はしない。

### アニメーション時間
- 画面遷移: 200ms ease-out (フェード+8px スライド)
- FAB拡大: 150ms
- 波形バー: 0.7–1.1s ループ(位相をずらす)
- 録音ドットパルス: 1.2s ループ
- AI処理スピナー: 1s linear 360°回転
- 保存チェックpop-in: 320ms、トースト表示1.5s後フェードアウト

### アイコン
Lucide Icons想定。線画(stroke 1.6–2px、丸端)。塗りつぶし/絵文字は不使用。サイズ16/18/20/24/34pxの5段階。

### タップ領域 / セーフエリア
全てのタップ可能要素は最低44×44pt。iOS Safe Areaは `env(safe-area-inset-top/bottom)` を利用し、特に録音FABとホームインジケータの干渉を回避する。
