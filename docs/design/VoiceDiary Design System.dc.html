ダークモードのルール: 単純な白黒反転ではなく、同一の暖色相を保持したまま明度を反転(背景 L≈97%→18%)。アクセント/録音色はダーク側で+8〜10%明度を上げ、暗背景でのコントラスト比を確保(本文テキストは背景に対しWCAG AA以上)。影は白背景では暖色寄りの薄い影、暗背景では黒の濃い影に切り替える。

2. 日本語フォント & タイプスケール

游ゴシック体の代替として Noto Sans JP を採用。行間は日本語の可読性を優先し広めに設定(1.6〜1.95)。
見出し Heading — 20–24px / 700 / line-height 1.4
画面タイトル・日記タイトル
小見出し Subheading — 16–17px / 600 / line-height 1.5
リスト項目タイトル・セクション見出し
本文 Body — 15–16px / 400 / line-height 1.85–1.95 — 日記本文や説明文に使用。既定の日本語読了速度を考慮し、1行40字前後で改行されるよう最大幅を制御する。
補足 Caption — 12–13px / 400 / line-height 1.6 — 日時・秒数・ヒント文言
フォントスタック: 'Noto Sans JP', -apple-system, system-ui, sans-serif ／ 等幅数値(タイマー等)には font-variant-numeric: tabular-nums

4. コンポーネント仕様

ボタン — Primary
高さ 52–56px・角丸 pill・背景 accent・文字 17px/600。1画面に1つのみ配置(強調は視覚的に一意)。フルwidth、親指到達域である画面下部に固定。押下時 scale(0.97)、150ms ease-out。
ボタン — Secondary / Icon
44×44pt以上を厳守。背景 card 色 or transparent、テキストのみ、または枠線1px border色。結果画面の二次操作(コピー・共有等)は横スクロールのチップ列で提示し、主要CTAと視覚的に競合させない。
録音ボタン (FAB)
76×76pt、画面下部中央・親指の自然な到達位置。タップで scale(1.08) の軽い拡大(150–200ms)。録音中は停止(角丸スクエア)・一時停止(バー2本 / 三角)・キャンセル(ゴミ箱)の3ボタン構成、中央の停止ボタンのみ大きく強調。
入力欄・折りたたみ
「元の文字起こし」はデフォルトで折りたたみ、タップで展開(chevron回転 200ms)。タイトル・本文は生成後インライン編集可能なテキストとして扱う(実装時は contentEditable もしくは textarea、フォーカス時にアクセント色の下線)。
アイコン
Lucide Icons を想定した線画(stroke 1.6–2px、丸端 round cap/join)。塗りつぶしアイコンや絵文字は使用しない。サイズは 16 / 18 / 20 / 24 / 34px の5段階。

5. アニメーション

画面遷移フェード+8px上方向スライド、200ms ease-out
録音開始FABが1.08倍に軽く拡大、150ms
録音中の波形各バーが0.7–1.1s周期でscaleY 0.4↔1をループ(位相をずらす)
録音中インジケータ赤ドットが1.2s周期でscale+opacityパルス
AI処理中円弧が1sで360°回転する穏やかなスピナーのみ。パーティクル等の演出は使用しない
保存完了チェックマークが 320ms でpop-in、トーストは1.5s表示後フェードアウト
Reduce Motion: OS設定の prefers-reduced-motion: reduce を検知した場合、上記すべてのループ・トランジションを無効化し瞬時切り替えに統一する(プロトタイプ上部の「動きを減らす」トグルで確認可能)。

6. 操作状態一覧

初回利用
オンボーディング → マイク権限確認の2ステップ。権限拒否時は理由と設定への導線を明示。
録音中/一時停止
タイマー・波形・赤ドットで状態を明示。一時停止中は波形静止・ドット灰色に変化。
長すぎる録音
目安20分(プロトタイプは20秒でデモ)超で警告バナーを表示、録音は継続可能。
文字起こし中/生成中
同一レイアウトでタイトル・本文コピーのみ切替。キャンセル可能。
通信エラー
文字起こし済みデータは保持されている旨を明記し、再試行のみで復帰できるようにする。
空の録音
2秒未満 or 無音判定時。再録音導線のみのシンプルなエラー。
日記生成完了
主要保存ボタンを1つだけ強調。折りたたみ文字起こし・コピー系・外部保存・書き直し・削除はチップ列に整理。
保存完了/未保存下書き
保存完了はトーストのみ(モーダルにしない)。未保存下書きはホーム上部にバナーで常時通知。

7. Claude Code ハンドオフ・メモ

構造方針: Next.js (App Router) + TypeScript + Tailwind CSS。トークンは tailwind.config.ts の theme.extend.colors に上記カラーを登録し(例: bg.DEFAULT / accent.DEFAULT / recording / warning / success)、ダークモードは class 戦略(dark:)で切替。
状態管理: 録音フローは有限状態機械として実装(idle → recording ⇄ paused → transcribing → generating → result / error / empty)。本プロトタイプの Component.state.screen がそのままステート名の参考になる。
アクセシビリティ: すべてのタップ領域は最低44×44pt。prefers-reduced-motion をCSSメディアクエリで検知し、Tailwindの motion-reduce: ユーティリティでアニメーションを無効化。
PWA要件: manifest.json でスタンドアロン表示・テーマカラーはライト/ダークそれぞれの背景色を指定。iOS Safe Area は env(safe-area-inset-*) をpadding-bottomに反映(ホームインジケータと主要ボタンの干渉回避)。
キーボード表示時: タイトル編集など入力を伴う画面では、主要保存ボタンをキーボード直上に追従させる(visualViewport APIでの高さ検知、もしくは sticky footer)。
